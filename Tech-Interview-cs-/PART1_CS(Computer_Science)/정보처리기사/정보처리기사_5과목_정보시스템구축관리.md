# 🌐 정보처리기사

## 5 정보시스템 구축관리

1. 구조적 방법론
   - 정형화된 분석 절차에 따라 사용자 요구사항을 파악하여 문서화하는 처리 중심 방법론
   - 복잡한 문제를 다뤄 분할과 정복 원리를 적용

2. 정보공학 방법론
   - 정보 시스템의 개발을 위해 정형화된 기법들을 상호 연관성 있게 통합 및 적용하는 자료 중심 방법론
   - 데이터 베이스 설계를 위한 데이터 모델링, 개체 관계도를 사용

3. 컴포넌트 기반 방법론
   - 기존의 시스템이나 소프트웨어를 구성하는 컴포넌트를 조합, 하나의 새로운 애플리케이션을 만드는 방법론
   - 분석 단계에서 사용자 요구사항, 정의서가 산출

4. 소프트웨어 재사용 이점
   - 개발시간과 비용 단축
   - 소프트웨어 품질 향상
   - 소프트웨어 개발 생산성 향상
   - 시스템 명세, 설계, 코드 등 문서 공유

5. 소프트웨어 재사용 방법
   - 합성 중심: 전자 칩과 같은 소프트웨어 부품, 블록을 만들어서 끼워 맞춰 소프트웨어 완성
   - 생성 중심: 추상화 형태로 쓰여 명세를 구체화하여 프로그램을 만드는 방법

6. 소프트웨어 재공학 이점
   - 위험 부담 감소
   - 개발 시간 단축
   - 개발 비용 절감
   - 시스템 명세의 오류 억제

7. 소프트웨어 재공학 주요 활동
   - 분석: 기존 소프트웨어 명세서 확인, 동작 이해, 재공학할 대상 선정
   - 재구성: 기존 소프트웨어 구조를 향싱시키기 위해 재구성
   - 역공학: 기존 소프트웨어 분석, 개발 과정과 데이터 처리 과정 설명하는 분석 및 설계 정보 재발견
   - 이식: 기존 소프트웨어를 다른 운영체제나 하드웨어 환경에 사용할 수 있도록 변환

8. CASE(Computer Aided Software Engineering)
   - 소프트웨어 개발 과정 전체 또는 일부를 컴퓨터와 전용 소프트웨어 도구를 사용하여 자동화하는것.

9.  CASE 기술
   - 구조적 기법
   - 프로토타이핑
   - 자동 프로그래밍
   - 정보 저장소
   - 분산처리

10. CASE 주요 기능
   -  소프트웨어 생명 주기 전 단계 연결
   -  다양한 소프트웨어 개발 모형 지원
   -  모델들의 모순 검사 및 오류 검증
   -  그래픽 지원
   -  자료 흐름도 작성

11.  비용 산정 기법 - LOC 기법
  - 소프트웨어 각 기능의 원시 코드 라인 수의 비관치, 낙관치, 기대치를 측정하여 예측치를 구하고 비용을 산정하는 기법

12.  LOC 기법 산정 공식
  - 노력 (개발기간 x 투입 인원), 1인당 월 평균 생산 코드 라인 수
  - 개발비용: 노력 x 단위 비용(1인당 웚 평균 인건비)
  - 개발기간: 노력 / 투입인원
  - 생산성: LOC / 노력

13. 수학적 산정 기법의 종류
  - COCOMO 모형
  - Putnam 모형
  - 기능 점수(Function Point) 모형

14. 비용 산정 기법 - Putnam 모형
  - 소프트웨어 생명 주기 전 과정 동안에 사용될 노력의 분포를 가정해주는 모형이다
  - Rayleigh-Norden 곡선 노력 분포도를 기초로 한다.

15. 비용 산정 기법 - COCOMO
  - 보햄이 제안한 것, 원시 프로그램의 규모에 의한 비용 산정 기법

16. COCOMO 소프트웨어 개발 유형
  - 조직행(Organic): 기관 내부에서 개발된 중소규모 소프트웨어, 일괄 자료 처리나 과학기술 계산용, 비즈니스 자료 처리용으로 5만 라인 이하 소프트웨어 개발
  - 반분리행(Semi-Detached): 조직형과 내장형의 중간형, 트랙잭션 처리 시스템이나 운영체제, 데이터 베이스 관리 시스템 등 30만 라인 이하 소프트웨어 개발
  - 내장형(Embedded): 내장형은 초대형 규모의 트랜잭션 처리 시스템, 운영체제 등의 30만 라인 이상의 소프트웨어를 개발

17. 자동화 추정 도구 - SLIM
  - Rayleigh-Norden 곡선과 Putnam 예측 모델 기초로 개발된 자동화 추정 도구.

18. 유스케이스 다이어그램 - 엑터(Actor)
  - 시스템과 상호작용을 하는 모든 외부 요소, 사람이나 외부 시스템을 의미한다.
  - 주엑터: 시스템을 사용함으로써 이득을 얻는 대상, 주로 사람이 해당한다
  - 부엑터: 주엑터의 목적 달성을 위해 시스템에 서비스를 제공하는 외부 시스템, 조직이나 기관 등이 될 수 있음.  

19. 순차(Sequenece) 다이어그램의 구성요소
  - 엑터(Actor)
  - 객체(Object)
  - 생명선(Lifeline)
  - 실행 상자(Active Box)
  - 메시지(Message)  

20. 사용자 인터페이스의 특징
  - 사용자의 편리성과 가독성을 높여준다.
  - 작업 시간을 단축시킨다.
  - 업우에 대한 이해도를 높여준다.
  - 사용자 중심으로 설계되어 있다.
