# 🌐 정보처리기사

## 5 정보시스템 구축관리

1. 구조적 방법론
   - 정형화된 분석 절차에 따라 사용자 요구사항을 파악하여 문서화하는 처리 중심 방법론
   - 복잡한 문제를 다뤄 분할과 정복 원리를 적용

2. 정보공학 방법론
   - 정보 시스템의 개발을 위해 정형화된 기법들을 상호 연관성 있게 통합 및 적용하는 자료 중심 방법론
   - 데이터 베이스 설계를 위한 데이터 모델링, 개체 관계도를 사용

3. 컴포넌트 기반 방법론
   - 기존의 시스템이나 소프트웨어를 구성하는 컴포넌트를 조합, 하나의 새로운 애플리케이션을 만드는 방법론
   - 분석 단계에서 사용자 요구사항, 정의서가 산출

4. 소프트웨어 재사용 이점
   - 개발시간과 비용 단축
   - 소프트웨어 품질 향상
   - 소프트웨어 개발 생산성 향상
   - 시스템 명세, 설계, 코드 등 문서 공유

5. 소프트웨어 재사용 방법
   - 합성 중심: 전자 칩과 같은 소프트웨어 부품, 블록을 만들어서 끼워 맞춰 소프트웨어 완성
   - 생성 중심: 추상화 형태로 쓰여 명세를 구체화하여 프로그램을 만드는 방법

6. 소프트웨어 재공학 이점
   - 위험 부담 감소
   - 개발 시간 단축
   - 개발 비용 절감
   - 시스템 명세의 오류 억제

7. 소프트웨어 재공학 주요 활동
   - 분석: 기존 소프트웨어 명세서 확인, 동작 이해, 재공학할 대상 선정
   - 재구성: 기존 소프트웨어 구조를 향싱시키기 위해 재구성
   - 역공학: 기존 소프트웨어 분석, 개발 과정과 데이터 처리 과정 설명하는 분석 및 설계 정보 재발견
   - 이식: 기존 소프트웨어를 다른 운영체제나 하드웨어 환경에 사용할 수 있도록 변환

8. CASE(Computer Aided Software Engineering)
   - 소프트웨어 개발 과정 전체 또는 일부를 컴퓨터와 전용 소프트웨어 도구를 사용하여 자동화하는것.

9.  CASE 기술
   - 구조적 기법
   - 프로토타이핑
   - 자동 프로그래밍
   - 정보 저장소
   - 분산처리

10. CASE 주요 기능
   -  소프트웨어 생명 주기 전 단계 연결
   -  다양한 소프트웨어 개발 모형 지원
   -  모델들의 모순 검사 및 오류 검증
   -  그래픽 지원
   -  자료 흐름도 작성

11.  비용 산정 기법 - LOC 기법
  - 소프트웨어 각 기능의 원시 코드 라인 수의 비관치, 낙관치, 기대치를 측정하여 예측치를 구하고 비용을 산정하는 기법

12.  LOC 기법 산정 공식
  - 노력 (개발기간 x 투입 인원), 1인당 월 평균 생산 코드 라인 수
  - 개발비용: 노력 x 단위 비용(1인당 웚 평균 인건비)
  - 개발기간: 노력 / 투입인원
  - 생산성: LOC / 노력

13. 수학적 산정 기법의 종류
  - COCOMO 모형
  - Putnam 모형
  - 기능 점수(Function Point) 모형

14. 비용 산정 기법 - Putnam 모형
  - 소프트웨어 생명 주기 전 과정 동안에 사용될 노력의 분포를 가정해주는 모형이다
  - Rayleigh-Norden 곡선 노력 분포도를 기초로 한다.

15. 비용 산정 기법 - COCOMO
  - 보햄이 제안한 것, 원시 프로그램의 규모에 의한 비용 산정 기법

16. COCOMO 소프트웨어 개발 유형
  - 조직행(Organic): 기관 내부에서 개발된 중소규모 소프트웨어, 일괄 자료 처리나 과학기술 계산용, 비즈니스 자료 처리용으로 5만 라인 이하 소프트웨어 개발
  - 반분리행(Semi-Detached): 조직형과 내장형의 중간형, 트랙잭션 처리 시스템이나 운영체제, 데이터 베이스 관리 시스템 등 30만 라인 이하 소프트웨어 개발
  - 내장형(Embedded): 내장형은 초대형 규모의 트랜잭션 처리 시스템, 운영체제 등의 30만 라인 이상의 소프트웨어를 개발

17. 자동화 추정 도구 - SLIM
  - Rayleigh-Norden 곡선과 Putnam 예측 모델 기초로 개발된 자동화 추정 도구.

18. 기능 점수(FP)모형 - 가중치 증대 요인
  - 자료 입력(입력 양식)
  - 정보 출력(출력 보고서)
  - 명령어(사용자 질의 수)
  - 데이터 파일
  - 필요한 외부 루틴과의 인터페이스

19. PERT(프로그램 평가 및 검토 기술)
  - 각 작업별로 낙관적인 경우, 가능성이 있는경우, 비관적인 경우로 나누어 각 단계별 종료 시기를 결정
  - 결정 경로, 작업에 대한 경계 시간, 작업 간의 상호 관련선 등을 알 수 있다.

21. 간트 차트(Gantt Chart)
  - 프로젝트 작업 일정을 막대 도표를 이용, 표시하는 프로젝트 일정표이다
  - 수평 막대의 길이는 각 작업의 기간을 나타낸다

22. 소프트웨어 프로젝트 관리
  - 주어진 기간 내 최소의 비용으로 사용자를 만족시키는 시스템을 개발하기 위한 전반적인 활동을 의미한다.

23. 위험 관리(Risk Analysis)
  - 프로젝트 추진 과정에서 예상되는 각종 돏발 상황을 미리 예상 이에 적절한 대책을 수립하는 일련의 활동

24. ISO/IEC 12207
  - 기본 생명 주기 프로세스: 획득, 공급, 개발, 운영, 유지 보수 프로세스
  - 지원 생명 주기 프로세스: 품질 보증, 검증, 확인, 활동 검토, 감사, 문서화, 형상 관리, 문제 해결 프로세스
  - 조직 생명 주기 프로세스: 관리, 기반 구조, 훈련, 개선 프로세스

25. CMMI의 소프트웨어 프로세스 성숙도 5단계
  - 초기(Initial)
  - 관리(Managed)
  - 정의(Defined)
  - 정량적 관리(Quantiatively Managed)
  - 최적화(Optimizing)

26. SPICE(소프트웨어 처리 개선 및 능력 평가 기준)
  - 소프트웨어 품질 및 생산성 향상을 위해 소프트웨어 프로세스를 평가 및 개선하는 국제 표준이다.

27. SPICE의 프로세스 수행 능력 단계
  - Level 0 - 불완전(Incomplete)
  - Level 1 - 수행(Performed)
  - Level 2 - 관리(Managed)
  - Level 3 - 확립(Established)
  - Level 4 - 예측(Predictable)
  - Level 5 - 최적화(Optimzing)

28. 소프트웨어 개발 방법론 테일러링
  - 프로젝트 상황 및 특성에 맞도록 정의된 소프트웨어 개발 방법론의 절차, 사용기법 등을 수정 및 보완하는 작업

29. 소프트웨어 개발 방법론 테일러링 고려사항
  - 내부적 기준
    - 목표 환경: 시스템의 개발 환경과 유형
    - 요구사항: 개발, 운영, 유지보수 등
    - 프로젝트 구모: 비용, 인력, 기간 등
    - 보유 기술: 프로세스, 개발 방법론, 산출물
  - 외부적 기준
    - 법적 제약사항: 프로젝트별로 적용 될 IT
    - 표준 품질 기준: 금융, 제도 등 분야별 표준 품질 기준
  
30. 소프트웨어 개발 프레임 워크
  - 소프트웨어 개발에 공통적으로 사용되는 구성 요소와 아키텍처를 일반화하여 손쉽게 구현할 수 있도록 여러가지 기능들을 제공해주는 반제품 형태의 소프트웨어 시스템이다.
  - 선행 사업자의 기술에 의존하지 않은 표준화된 개발 기반으로 인해 사업자 종속성이 해소된다.
  - 개발자가 관리하고 통제해야 하는 객체들의 제어를 프레임워크에 넘겨 생산성을 향상시킨다. 

31. SDN(Software Defined Networking)
  - 네트워크를 컴퓨터처럼 모델링, 여러 사용자가 각각의 소프트웨어들로 네트워킹을 가상화하여 제어하고 관리하는 네트워크다.

32. SDS(Software-Defined Storage)
  - 물리적인 데이터 스토리지(Data Storage)를 가상화하여 여러 스토리지를 하나처럼 관리하는 기술이다.

33. SDDC(Software Defined Data Center)
  - 데이터 센터의 모든 자원을 가상화하여 인력의 개입없이 소프트웨어 조직만으로 관리 및 제어되는 데이터 센터이다.

34. 메시 네트워크(Mesh Network)
  - 수십에서 수천 개의 디바이스를 구물망(Mesh)과 같이 유기적으로 연결, 모든 구간을 동일한 무선망처럼 구성하여 사용자가 안정적인 네트워크를 사용할 수 있게 한다.

35. 피코넷(PICONET)
  - 여러 개의 독립된 통신장치가 블루투스 기술이나 UWB 통신 기술을 사용하여 통신망을 형성하는 무선 네트워크 기술이다.

36. 클라우드 기반 HSM
  - 클라우드를 기반으로 암호화 키의 생성, 저장, 처리 등의 작업을 수행하는 보안기기를 가리키는 용어다.
  - 암호화 키 생성이 하드웨어적으로 구현되어 소프트웨어적으로 구현된 암호 기술이 가지는 보안 취약점을 무시할 수 있다.

37. 파스-타(PaaS-TA)
  - 소프트웨어 개발 환경을 제공하기 위해 개발한 개방형 클라우드 컴퓨터 플랫폼이다.

38. 징(Zing)
  - 10cm 이내 거리에서 3.5Gbs 속도의 데이터 전송이 가능한 초고속 근접무선통신(NFC)이다.

39. SSO(Single Sign On)
  - 한 번의 로그인으로 개인이 가입한 모든 사이트를 이용할 수 있게 해주는 시스템이다.

40. 스마트 그리드(Smart Grid)
  - 전력선을 기반으로 모든 통신, 정보, 관련 애플리케이션 인프라를 하나의 시스템으로 통합하여 관리함으로써 효율적인 에너지 관리가 가능하다.

41. WDW(WaveLength Division Multiplexing)
  - 파장이 다른 광선끼리 서로 간섭을 일으키지 않는 성질을 이용하여 여러 대의 단말기가 동시에 통신 회선을 사용할 수 있도록 하는 기술이다.

42. 버스형(Bus)
  - 한 개의 통신 회선에 여러 대의 단말장치가 연결된 형태이다.

43. VLAN(Virtual Local Area Network)
  - LAN의 물리적인 배치와 상관없이 논리적으로 분리하는 기술, 접속된 장비들의 성능 및 보완성을 향상시킬 수 있다.

44. WPA(Wi-Fi Protected Access)
  - Wi-Fi에서 제정한 무선 랜(WLAN)인증 및 암호화 관련 표준이다.

45. CSMA/CA
  - 무선 랜에서 데이터 전송 시 매체가 비어있음을 확인한 뒤 충돌을 피하기 위해 일정한 시간을 기다린 후 데이터를 전송하는 방법이다.
