## 1. HTTP란?

HTTP = HyperTextTransferProtocol

하이퍼텍스트를 전송하는규칙이다.
처음엔 HTML 문서만 주고받기 위해 만들어졌으나, 지금은 인터넷에서 데이터를 주고받는 거의 모든것에 사용된다.

**HTTP로 전송할 수 있는 것들**
| 종류 | 예시 |
|------|------|
| 문서 | HTML, TEXT |
| 미디어 | 이미지, 음성, 영상, 파일 |
| 데이터 | JSON, XML (API 통신) |
| 기타 | 거의 모든 형태의 데이터 |

> 유튜브 영샹을 보든, 카카오톡 메시지를 보내든, 쇼핑몰에서 주문을 하든 전부 **HTTP**가 사용될 확률이 높다.

## 2. HTTP의 역사

```
1991년  HTTP/0.9  ── GET 메서드만 존재. 정말 기초적인 버전.
  │
1996년  HTTP/1.0  ── 메서드와 헤더가 추가됨.
  │
1997년  HTTP/1.1  ── 현재까지 가장 많이 쓰이는 버전 ⭐
  │                  (이 버전을 기준으로 공부하면 됩니다!)
  │
2015년  HTTP/2    ── 속도 성능 개선
  │
현재    HTTP/3    ── TCP 대신 UDP 사용, 더 빠른 성능 목표
```

- HTTP/1.1 HTTP/2-> TCP 기반
- HTTP/3->UDP 기반

## 3. 클라이언트 서버 구조

HTTP는 요청(Request)->응답(Response)구조로 작동한다.

```
클라이언트                        서버
   │                               │
   │  ① 요청 (Request)             │
   │ ─────────────────────────────>│
   │                               │ (결과를 만들어냄)
   │  ② 응답 (Response)            │
   │ <─────────────────────────────│
```

- 클라이언트 = 요청하는 쪽(브라우저, 앱 등)
- 서버 = 요청을 받아 처리하고 응답을 돌려주는 쪽

클라이언트와 서버를 분리하면 각자의 역할에만 집중할 수 있다.

- 클라이언트 -> UI/사용성에 집중
- 서버->데이터 처리/비지니스 로직에 집중

## 4. Stateful, Stateless

서버가 클라이언트의 상태를 기억하냐, 안하냐의 차이이다.

---

**Stateful(상태유지)**
서버가 이전 대화 내용을 기억한다.

```
고객: 이 노트북 얼마인가요?
점원: 100만원입니다.               ← "노트북" 기억

고객: 2개 구매하겠습니다.
점원: 200만원입니다. 카드/현금?    ← "노트북 2개" 기억

고객: 신용카드로 구매할게요.
점원: 200만원 결제 완료!
```

**점원이 중간에 바뀌면?**

```
고객: 2개 구매하겠습니다.
점원B: ❓ 무엇을 2개 구매하시겠어요?  ← 기억 못함!
```

> ⚠️ 항상 **같은 서버**가 응답해야 합니다.  
> 그 서버에 장애가 나면 대화가 처음부터 다시 시작됩니다.
