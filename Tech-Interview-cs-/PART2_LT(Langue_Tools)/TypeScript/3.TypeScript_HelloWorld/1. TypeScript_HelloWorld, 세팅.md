# 🌐 TypeScrpt 실행, 컴파일러 설치, 컴파일러 옵션 설정

## 1. 타입스크립트 세팅

### 1_1. Node.js패키지 초기화

- 방금 만든 폴더에 Node.js 패키지를 초기화한다.

```bash
npm init
```

- 패키지 정보는 모두 default로 설정한다. 패키지 기본 정보를 자세히 설정할 필요가 없다.

### 1-2. @types/node 설치

- 타입스크립트 프로젝트를 시작할 때 패키지 초기화 이후 @types/node라는 패키지를 설치해야 한다.

```bash
npm i @types/node
```

- @types/node 라이브러리는 Node.js가 제공하는 기본 기능에 대한 타입 정보를 가지고 있다.
- Node.js가 제공하는 console등의 기본기능 들의 타입이 선정되지 않아 컴파일 과정에서 타입 검사가 실패하여 오류가 발생할 수 있다.
  > ### 💡 이 패키지를 꼭 설치해야 하는 이유
  >
  > **타입스크립트**는 코드를 실행하기 전에 타입을 올바르게 사용했는지 검사하는 **‘타입 검사’** 과정을 거칩니다.
  >
  > 이 검사 과정에서 타입이 선언되지 않은 코드를 만나게 되면, 타입스크립트는 타입이 올바르게 사용 되지 않았다고 생각해 **오류를 발생** 시킵니다.
  >
  > 따라서 Node.js의 기본 기능들을 위한 타입을 별도로 선언하기 위해 `@types/node` 패키지를 설치해야 합니다.

### 1-3. TSX(TypeScript Execute)로 실행

- 명령어 한번으로 타입스크립트 코드를 실행켜주는 도구 tsx(TypeScript Execute)를 확인한다.
- tsx는 한 번의 명령어로 타입스크립트 코드를 실행시켜주는 도구다. 타입스크립트를 설치할 때와 동일하게 npm에 -g 옵션을 붙여 글로벌로 설치한다.

```bash
# mac일 경우
sudo npm i -g tsx

# win일 경우
npm i -g tsx
```

- 설치가 완료되면 명령어를 제대로 ㅇ설치 되었는 지 확인한다.

```bash
tsx -v
tsx v4.6.2
node v20.9.0
```

- 설치가 완료되었다면 tsx를 이용해 타입스크립트 파일을 실행한다.

```bash
tsx src/index.ts
# 명령어 답
Hello TypeScript
```

- tsx를 이용하면 한 번의 명령어로 타입스크립트를 실행할 수 있다.
  > ### 💡 TSX 사용을 권장합니다 (Node.js 20 버전에서는 동작하지 않습니다)
  >
  > 23년 10월 Node.js의 **LTS(장기 지원 버전)**가 20대로 업데이트 되면서 `ts-node`가 정상적으로 동작하지 않고 있습니다.
  >
  > 따라서 23년 10월 이후에 이 강의를 수강하시는 분들께서는 `ts-node` 대신 위에서 소개하는 **`tsx`**를 사용하시기 바랍니다.
