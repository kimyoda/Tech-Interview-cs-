# 🌐 Git cherry-pick으로 선택된 커밋만 개발 브랜치에 적용하기

> 전체 브랜치 머지 없이 특정 커밋만 골라 다른 브랜치에 반영하고 싶을 때 `git cherry-pick`을 활용한다.
> 해당 글에서는 실제 기능 개발 중 겪은 상황을 예시로, cherry-pick의 사용법과 주의 점을 정리하였다.

---

## 1. 배경

기능을 개발하던 중, `feature/example_contents` 브랜치의 **일부 커밋**만 `develop` 브랜치에 먼저 반영해야 하는 상황이 생겼다.
브랜치를 전체를 머지하면 **아직 작성중이거나 잠시 푸쉬한 코드**까지 함께 들어간다는 점이었다.
이때 유효한 해결책이 바로 `git cherry-pick`이다.

> 공식 문서 정의
> `git cherry-pick`은 기존 커밋이 도입한 변경을 현재 브랜치에 하나씩 적용해 **새로운 커밋을 만드는 명령**이다.
> 한 브랜치의 특정 커밋을 골라 다른 브랜치에 적용하는 행위이다.

---

## 2. 이슈 상황

반영이 필요한 커밋은 아래 세게였다.

| 커밋 해시  | 설명                                                                               |
| ---------- | ---------------------------------------------------------------------------------- |
| `a0efa0ab` | `[Feature]` 미니게임 시작 관련 프로토콜, 응답, 테스트 코드 추가 및 수정            |
| `a6c36987` | `[Refactor]` 미니게임 시작 핸들러 수정                                             |
| `00542c6e` | `[Refactor]` 미니게임 스타트 프로토콜, 서비스, 테스트 코드 클라 응답값에 맞춰 변경 |

나머지 커밋들은 아직 작업 중인 상태였기 때문에 **전체 브랜치 머지는 어려운**한 이슈였다.
