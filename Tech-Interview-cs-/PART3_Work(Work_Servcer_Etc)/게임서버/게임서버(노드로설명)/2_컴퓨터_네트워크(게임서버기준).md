# 2장 컴퓨터 네트워크 - 네트워크 기초

---

## 1. 컴퓨터 네트워크를 구성하는 기기

네트워크는 컴퓨터들을 연결하는 인프라이다. 주요 구성 기기는 다음과 같다

| 기기                            | 역할                                           | OSI 계층            |
| ------------------------------- | ---------------------------------------------- | ------------------- |
| **허브(Hub)**                   | 연결된 모든 포트에 데이터 전달 (브로드 캐스트) | 계층 1 (물리)       |
| **스위치(Switch)**              | MAC 주소 기반으로 특정 포트에만 데이터 전달    | 계층 2 (데이터링크) |
| **라우터(Router)**              | IP 주소 기반으로 최적 경로를 찾아 패킷 전달    | 계층 3 (네트워크)   |
| **NIC(Network Interface Card)** | 컴퓨터를 네트워크에 연결하는 하드웨어          | 게층 1~2            |

```
일반적인 네트워크 구성

인터넷
  ↕
라우터 (ISP 제공 / 공유기)
  ↕
스위치 (사무실/데이터센터 내부망)
  ↕  ↕  ↕
 PC  서버  서버

```

## 2. OSI 모델

**OSI(Open Systems Interconnection) 모델**은 네트워크 통신을 7개 계층으로 나눠 표준화한 모델이다. 각 계층은 하위 계층의 서비스를 이용하고 상위 계층에 서비스를 제공한다.

```
7. 응용 계층 (Application) - HTTP, WebSocket, gPRC
6. 표현 계층 (Presentation) - 데이터 인코딩, 암호화(LTS)
5. 세션 계층 (Session) - 연결 관리, 세션 제어
4. 전송 계층 (Transport) - TCP, UDP - 포트 번호
3. 네트워크 계층 (Network) - IP, 라우팅 IP주소
2. 데이터링크 계층 (Data Link) - 이더넷, MAC 주소
1. 물리 계층 (Physical) - 케이블, 전기 신호
```

게임 서버 개발자가 주로 다루는 계층은 **계층 3(IP)**과 **계층 4(TCP/UDP)**이다. HTTP/WebSocket은 계층 7에 해당한다.

### OSI 계층 2 - 데이터링크 계층

이더넷이 여기에 속한다. **MAC 주소(Media Access Control Address)**를 사용해 같은 네트워크(LAN) 내의 기기를 식별한다.

- MAC 주소: 48비트, 하드웨어에 고정된 고유 주소
- 같은 스위치에 연결된 기기 간 통신에 사용
- 라우터를 넘어가면 MAC 주소가 변경된다 (IP 주소는 유지한다)

### OSI 계층 3- 네트우 ㅓ크 계층

**IP(Internet Protocol)**이 여기에 속한다. 서로 다른 네트워크를 연결하고 **라우팅(최적 경로 탐색)**을 담당한다.

- IPv4: 32비트 주소(예: `192.168.0.1`)
- IPv6: 128비트 주소(예: `2001:0db8:85a3::8a2e:0370:7334`)

## 3. 인터넷

**인터넷**은 전 세계의 수많은 네트워크를 하나로 연결한 거대한 네트워크이다. IP 프로토콜을 공통 언어로 사용한다.

```
인터넷 구조 - 단순화
한국 ISP 망 ──── 인터넷 백본(광케이블) ──── 미국 ISP 망
     ↕                                          ↕
내 컴퓨터                                   미국 서버
```

게임 서버는 데이터센터(AWS, GCP, Azure 등)에 위치, 플레이어는 인터넷을 통해 연결된다. 이 구간이 레이턴시(지연 시간)가 게임 품질에 영향을 미친다.
