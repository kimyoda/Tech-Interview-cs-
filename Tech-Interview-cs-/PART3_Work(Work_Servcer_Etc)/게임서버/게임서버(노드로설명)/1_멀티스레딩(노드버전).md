# 1장 멀티스레딩 - 게임서버 개발자가 알아야할 동시성 기초

## 1. 프로그램과 프로세스

**프로그램(Program)**은 디스크에 저장된 실행 파일이다. 이것이 메모리에 올라가 실행되면 **프로세스(Process)**가 된다.
프로세스는 운영체제로부터 독립된 메모리 공간(코드, 데이터, 힙, 스택)을 할당받는다.
프로세스 간에는 기본적으로 메모리를 공유 하지 않기에, 서로 간섭하지 않고 안전하게 실행된다.

```
[프로그램 실행 흐름]

실행 파일(.exe / 바이너리)
        ↓  OS가 메모리에 로드
     프로세스
    ├── 코드 영역  (명령어)
    ├── 데이터 영역 (전역/정적 변수)
    ├── 힙 영역   (동적 할당 메모리)
    └── 스택 영역  (함수 호출, 지역 변수)
```

## Node.js에서 프로세스

Node.js에 `child_process` 모듈이나 `cluster` 모듈로 복수의 프로세스를 만들 수 있다.
각 프로세스는 독립된 메모리를 가지며, IPC(프로세스 간 통신)로 데이터를 주고 받는다.

```js
// cluster 모듈로 멀티 프로세스 구성
const cluster = require("cluster");
const os = require("os");

if (cluster.isPrimary) {
  const numCPUs = os.cpus().length;
  console.log(`Primary ${process.pid} 실행 중, CPU: ${numCPUs}개`);

  for (let i = 0; i < numCPUs; i++) {
    cluster.fork(); // 워커 프로세스 생성
  }

  cluster.on("exit", (worker) => {
    console.log(`Worker ${worker.process.pid} 종료, 재시작`);
    cluster.fork();
  });
} else {
  // 워커에서 Express/NestJS에서 실행
  require("./app").listen(3000);
  console.log(`Worker ${process.pid} 시작`);
}
```

## 스레드

**스레드(Thread)**는 프로세스 안에서 실행되는 실행 흐름의 단위이다. 하나의 프로세스는 최소 1개의 스레드(메인 스레드)를 가지며, 여러 스레드를 만들어 동시에 작업을 처리할 수 있다.

프로세스, 스레드 차이
| 구분 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 (공유 없음) | 힙/데이터 영역 **공유** |
| 생성 비율 | 높음 | 낮음|
| 통신 방식 | IPC(소켓, 파이프 등) | 공유 메모리에 직접 접근 |
| 오류 격리 | 한 프로세스 오류가 다른 프로세스에 영향이 없음 | 한 스레드 오류가 전체 프로세스에 영향 |
| 문맥 교환 비용| 높음 | 낮음 |
