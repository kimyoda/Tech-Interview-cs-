# 1장 멀티스레딩 - 게임서버 개발자가 알아야할 동시성 기초

## 1. 프로그램과 프로세스

**프로그램(Program)**은 디스크에 저장된 실행 파일이다. 이것이 메모리에 올라가 실행되면 **프로세스(Process)**가 된다.
프로세스는 운영체제로부터 독립된 메모리 공간(코드, 데이터, 힙, 스택)을 할당받는다.
프로세스 간에는 기본적으로 메모리를 공유 하지 않기에, 서로 간섭하지 않고 안전하게 실행된다.

```
[프로그램 실행 흐름]

실행 파일(.exe / 바이너리)
        ↓  OS가 메모리에 로드
     프로세스
    ├── 코드 영역  (명령어)
    ├── 데이터 영역 (전역/정적 변수)
    ├── 힙 영역   (동적 할당 메모리)
    └── 스택 영역  (함수 호출, 지역 변수)
```

---

## Node.js에서 프로세스

Node.js에 `child_process` 모듈이나 `cluster` 모듈로 복수의 프로세스를 만들 수 있다.
각 프로세스는 독립된 메모리를 가지며, IPC(프로세스 간 통신)로 데이터를 주고 받는다.

```js
// cluster 모듈로 멀티 프로세스 구성
const cluster = require("cluster");
const os = require("os");

if (cluster.isPrimary) {
  const numCPUs = os.cpus().length;
  console.log(`Primary ${process.pid} 실행 중, CPU: ${numCPUs}개`);

  for (let i = 0; i < numCPUs; i++) {
    cluster.fork(); // 워커 프로세스 생성
  }

  cluster.on("exit", (worker) => {
    console.log(`Worker ${worker.process.pid} 종료, 재시작`);
    cluster.fork();
  });
} else {
  // 워커에서 Express/NestJS에서 실행
  require("./app").listen(3000);
  console.log(`Worker ${process.pid} 시작`);
}
```

## 스레드

**스레드(Thread)**는 프로세스 안에서 실행되는 실행 흐름의 단위이다. 하나의 프로세스는 최소 1개의 스레드(메인 스레드)를 가지며, 여러 스레드를 만들어 동시에 작업을 처리할 수 있다.

프로세스, 스레드 차이
| 구분 | 프로세스 | 스레드 |
|------|----------|--------|
| 메모리 공간 | 독립적 (공유 없음) | 힙/데이터 영역 **공유** |
| 생성 비율 | 높음 | 낮음|
| 통신 방식 | IPC(소켓, 파이프 등) | 공유 메모리에 직접 접근 |
| 오류 격리 | 한 프로세스 오류가 다른 프로세스에 영향이 없음 | 한 스레드 오류가 전체 프로세스에 영향 |
| 문맥 교환 비용| 높음 | 낮음 |

## Node.js에서 스레드 - Worker Threads

Node.js는 기본적으로 **싱글스레드 이벤트 루프**모델이나, `worker_threads` 모듈로 진짜 멀티스레드를 사용할 수 있다.

```js
// main.js
const { Worker, isMainThread, parentPort, workerData } = require('worker_threads');

if (isMainThread) {
  // 메인스레드: 워커 생성
  const worker = new Worker(__filename, {
    workerData: { taskId: data: [1, 2, 3, 4, 5]}
  });

  worker.on('message', (result) => {
    console.log('워커 결과:', result); // 워커 결과
  });

  worker.on('error', (err) => console.log('워커 오류:', err));
  worker.on('exit', (code) => console.log(`워커 종료 코드: ${code}`));
} else {
  // 워커 스레드: 무거운 연산 처리
  const { data } = workerData;
  const sum = data.reduce((acc, n) => acc + n, 0);
  parentPort.postMeessag(sum);
}
```

> 스레드는 같은 프롯세스의 힙 메모리를 공유하기 때문에 빠르나, 그만큼 동기화 문제(레이스 컨디션, 교착상태)에 주의해야 한다

## 멀티스레드 프로그래밍은 언제?

멀티스레드가 항상 답은 아니다. 다음 세가지 상황에서 효과적이라고 생각한다

### 오래 걸리는 업무 1개, 빨리 끝나는 일 여럿을 동시에 처리할 때

게임 서버에서 DB 저장(오래 걸림)을 하며 동시에 네트워크 패킷(빨리 끝남)을 계속 처리해야 할 때

```
싱글스레드
패킷 처리 -> DB저장(대기...) -> 패킷 처리 -> DB저장(대기..) <- 해당 구간 동안 패킷 처리 불가

멀티스레드
스레드1: 패킷처리 -> 패킷처리 -> 패킷처리 (계속 진행)
스레드2:           DB저장 -> DB저장 (별도 처리)
```

### 긴 처리를 진행하는 동안 다른 짧은 일을 처리할 때

게임 서버에서 레벨 데이터 로딩처럼 오래 걸리는 초기화 작업 중에도 로비 채팅은 계속 동작해야 하는 경우다.

### CPU 코어를 모두 활용해야 할 때

CPU 집약적 연산(물리 계산, 경로 탐색, 매치메이킹 알고리즘 등)을 멀티코어로 병렬 처리한다.

```ts
// NestJS: CPU 집약적 작업을 Worker Thread로 분리
import { Injectable } from "@nestjs/common";
import { Worker } from "worker_threads";
import * as path from "path";

@Injectable()
export class PathfindingService {
  findPath(mapData: number[][], start: Point, end: Point): Promise<Point[]> {
    return new Promise((resolve, reject) => {
      const worker = new Worker(
        path.resolve(__dirname, "pathfinding.worker.js"),
        { workerData: { mapData, start, end } },
      );

      worker.once("message", resolve);
      worker.once("error", reject);
    });
  }
}
```

> Node.js경우: I/O작업(DB, 네트워크, 파일)은 비동기 이벤트 루프가 이미 효율적으로 처리한다. Worker Threaeds는 주로 CPU 집약적 연산에 사용한다.
