# 게임서버*API*명세서를*잘쓰는*요령

> API 명세서가 클라와의 계약서라고 생각한다. 대충쓰면 나한테 다시 돌아온다.

---

## 1. 작성계기

게임서버 개발자로 일하다보면 하루에 API를 3~5개 뽑아내야 할 때가 있다.
랭킹, 상점, 이어하기, 이벤트, 시작하기, 아이템 구매, 도전횟수갬 등 기능이 복잡할수록 명세서를 어떻게 쓰느냐가 전체 개발 속도를 좌우한다.
해당 글에서 내가 실제로 사용하는 방식을 예제 기반으로 공유하고자 한다.

---

## 1. API 명세서의 기본구조

명세서는 크게 세 가지 분류이다.

```
1. 개요 (method/ endpoint / 한 줄 설명)
2. 요청 (Request Parameter)
3. 응답 (Response Field)
4. 처리 내용 (서버 로직 흐름)
5. 생플 응답 (실제 테스트 데이터 기반)
```

각 파트를 어떻게 쓴든 지, 실제 예제로 확인한다.
